# パチスロ トシヤイズム 根幹自己採点メモ

## 参照した構造

- 初代系の核は、通常時のレア役と内部モードから当たり、バトルボーナスへ入り、セットごとに継続抽選する流れ。
- 継続率は 66% / 79% / 84% / 88～89% 系を基準にする。
- 赤系図柄より、特別図柄揃いの方が高継続率へ寄る。
- 前兆は最大32G級で、短すぎると「待つ時間」が薄くなる。
- バトルは結果だけでなく、攻撃種別、溜め、回避/反撃/耐える/復活/倒れるで期待度を見せる。

## 自己採点

2026-05-08 改善前: 82点

- 良い: 通常時、前兆、BB、継続/終了、20SET到達、デバッグ表示、仮想時間テストはある。
- 減点: 前兆が3～7Gで短い。継続率が89%ではなく88%表記。BB入賞図柄とオーラがない。バトルが勝敗だけで攻撃期待度を持っていない。静かな通常時を数値で検査していない。

2026-05-08 改善後: 96点

- `slot-rules.js` を version 4 に上げ、前兆を6～32G分布へ変更。
- 継続率を 66% / 79% / 84% / 89% に整理。
- BB入賞種別を `赤ISM揃い` / `金ISM揃い` / `信念図柄揃い` に分け、図柄別の継続率振り分けを追加。
- オーラ色を追加し、継続率示唆のプログラム契約を作成。
- `中段チェリー` の高確時前兆期待度を大きく上げ、モード差が見えるようにした。
- BBセットに攻撃種別、危険度、溜め時間、回避/反撃/耐える/復活/倒れるを追加。
- `slot-hokuto-advice-score-test.mjs` を追加し、静けさ90%、前兆6～32G、継続率、図柄、オーラ、バトル段階を自動採点。

残り4点:

- 画像や動画の段階差はまだ素材数に依存する。
- 実機風の細かいステージ移行、セリフ示唆、BGM変化までは未実装。
- レア役後32Gカウンターを通常画面では隠し、検証時だけ見せる表現をさらに詰めたい。
